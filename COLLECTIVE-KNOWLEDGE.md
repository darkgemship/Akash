# Data Qi — Trí tuệ tập thể: 3 tầng & cơ chế "Bắn qua"

> Tầm nhìn: một Wikipedia sống. Trải nghiệm cá nhân chảy lên làm giàu tri thức tập đoàn, rồi vươn ra nhân loại. Mọi người cùng xây.

## 1. Dòng chảy 3 tầng (propagation)
```
KHO CÁ NHÂN (tự do viết)
   daily note · emotion · trải nghiệm↔bài học · câu chuyện đời · kỹ năng đã xây
      │ ① BẮN QUA: đề xuất đóng góp → [Tổng biên tập duyệt + check link]
      ▼
KHO TẬP ĐOÀN (canon QNET — chỉ Editor up; user comment/đề xuất)
      │ ② BẮN QUA: tri thức phổ quát "tốt nghiệp" → [duyệt nhẹ/cộng đồng]
      ▼
KHO NHÂN LOẠI (wiki mở — mọi user xây thêm)
```
**Cross-layer edges** = sợi nối "bắn qua" (hiển thị trên đồ thị 3 lớp).

## 1b. Phân cấp & cơ chế BẮN node-to-node (đã chốt)
**Phân cấp:** `KHO → 📂 Bài học lớn (folder) → 📄 Note/page`. Độ Thấm đo ở **folder**.

```
① NEO    — bài học CÁ NHÂN neo vào 1 bài học TẬP ĐOÀN (vd "Kỷ luật của tôi" → canon "Lãnh đạo bản thân")
② CHÍN   — folder cá nhân đạt Độ Thấm cao (Evidence+Experience+Action)
③ CHO ĐI — user bấm Transmission → tạo "HẠT TRI THỨC" (insight/bằng chứng/trải nghiệm ĐÃ CHƯNG CẤT, không phải note thô)
④ DUYỆT  — Tổng biên tập duyệt hạt
⑤ ĐÍNH   — hạt gắn vào node tập đoàn + vẽ cạnh cross-layer → 💫 TIA chạy dọc cạnh
⑥ LỚN    — node tập đoàn gom hạt từ NHIỀU người → Độ Thấm tập thể tăng → sáng & to hơn
⑦ TỐT NGHIỆP — bài học đủ phổ quát → curator bắn lên NHÂN LOẠI
```
**Cái "bắn" giữa 2 node = HẠT đã chưng cất**, KHÔNG phải nguyên trang. Note thô ở lại vault cá nhân.

**3 loại cạnh:** intra-layer (knowledge link 6 chiều) · anchor (cá nhân↔canon) · contribution (hạt đã duyệt chảy lên = tia bắn).

## 1c. NODE ĐỐI ỨNG — cách cá nhân nối vào canon (đã chốt)
Khi user ghi trải nghiệm/cảm xúc/bài học về 1 bài canon → nó bắn vào **NODE ĐỐI ỨNG CÁ NHÂN**
(bản sao cá nhân của bài học đó), KHÔNG đính thẳng vào canon.

```
CANON (khách quan, bất biến)        ĐỐI ỨNG (chủ quan, lớn dần)
📁 Lãnh đạo bản thân ◄──anchor──► 📂 "Lãnh đạo bản thân (của TÔI)"
  • block "kỷ luật là tự do" ◄experience-of┐  ├ 📝 trải nghiệm 😌
                                            ├ 📝 reflection
                                            └ ⚡ action log
                                      ▲ Độ Thấm radar Ở ĐÂY (node đối ứng)
```
- 2 loại tri thức (constructivism): **canon** = QNET dạy · **đối ứng** = user kiến tạo. Nối qua cạnh `anchor`.
- Cảm xúc = chiều `emotion` (tag) → nuôi tham số Meaning.

### 3 quy tắc
1. **Tạo LƯỜI:** chỉ tạo node đối ứng khi user thật sự phản tư (KHÔNG mirror cả cây canon).
2. **Neo theo độ mịn:** cả bài → neo folder; 1 ý → neo block (block-level ID).
3. **Node đối ứng = TÍCH HỢP:** neo nhiều canon + nhân loại cùng lúc → nơi canon + nhân loại + đời user hội tụ.

### Vì sao tối ưu
- Độ Thấm sống ở node đối ứng (đo mức nhập tâm = chuyện cá nhân).
- Backlink 2 chiều: canon↓ thấy trải nghiệm cộng đồng; đối ứng↑ thấy nguồn nuôi.
- "Bắn lên": chỉ node đối ứng đã chín mới đóng góp lên canon.

## 2. Quản trị từng tầng (Wikipedia có kiểm soát)
| Tầng | Tạo gốc | User thường | Cổng duyệt |
|------|---------|-------------|-----------|
| Cá nhân | chính chủ | viết tự do | — |
| Tập đoàn | **chỉ Editor** | comment, đề xuất link, chia sẻ trải nghiệm | **Tổng biên tập** duyệt link + kiểm đúng đắn |
| Nhân loại | **mọi user** | thêm khóa học/kiến thức | **duyệt nhẹ bởi user điểm Trí Tuệ cao** (như Wikipedia editor) |

**Ghi danh khi đóng góp (đã chốt):** user **TỰ CHỌN** ghi danh hay ẩn danh cho từng đóng góp được "bắn lên" tập đoàn.
