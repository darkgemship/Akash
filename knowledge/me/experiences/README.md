# experiences/

Nơi `/improve-system` lưu **bài học và câu chuyện** bạn chia sẻ trong các phiên làm việc.

Mỗi file = 1 trải nghiệm, đặt tên `YYYY-MM-DD-slug.md`, dùng `event_date` thật (ngày chuyện xảy ra, không phải ngày tạo file). Frontmatter `date` + `tags`, thân bài gồm bối cảnh → **Bài học** → **Áp dụng**.
